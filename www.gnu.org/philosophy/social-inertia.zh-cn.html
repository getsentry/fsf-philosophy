克服社会惯性

Richard Stallman 著

距离 GNU 和 Linux 组合第一次让自由使用个人电脑成为可能已经过去将近 20 年了。我们经历了漫长的旅程。现在你甚至可以从不止一家硬件销售商那里买到预装 GNU/Linux 的笔记本电脑——虽然他们销售的系统还不是完全的自由软件。那么是什么阻止我们取得完全的成功呢？

阻碍软件自由取得胜利的主要障碍是社会惯性。它有多种形式，你们肯定看到过不少。具体例子包括只在 Windows 系统下工作的设备，只能通过 Windows 系统访问的商业网站，只在 Windows 下运行的 BBC iPlayer“带手铐软件”。如果你更看重短期便利而不是自由，那么你可能会觉得这些是使用 Windows 的理由。因为大多数公司目前使用 Windows，所以考虑当下利益的学生希望学习如何使用 Windows 并请求学校教授这些内容。学校教授 Windows，毕业生习惯于使用 Windows，这就鼓励商业环境使用 Windows。

微软积极培育这个惯性：它鼓励学校灌输对 Windows 的依赖，并承接网站建设，导致这些网站只在 IE 浏览器下可用。

数年前，微软广告说运行 Windows 比运行 GNU/Linux 省钱。他们的比较已被真相揭穿，但是他们的论点有值得注意的深层次漏洞，这就是其中隐含的一种社会惯性：“目前，了解 Windows 的技术人员比 GNU/Linux 更多。”看重自由的人不会因为省钱就放弃自由，但是许多商业主管坚信任何所有物都是资产的理念，即便是自由，也可以用来销售。

社会惯性是由向社会惯性投降的人组成的。当你屈服于社会惯性，那么你就成了向他人施压的那部分；当你反抗时，你就减少了压力。我们通过识别社会惯性、并决定不成为其中一份子来克服社会惯性。

有一个弱点阻拦着社区：大多数 GNU/Linux 用户从未听说过激励 GNU 开发的自由理念，所以他们还是靠着短期的便利而不是靠自由来判断事物。这就使他们容易被社会惯性牵着鼻子走，以至于变成社会惯性的一部分。

要建立社区的抵抗力，我们需要谈论自由软件和自由——而不仅仅是开源支持者挂在嘴边的实际好处。我们需要通过 尽可能采取行动 并大声说出我们的道理来抵抗非自由软件。当越来越多的人认识到我们应该采取什么行动来克服惯性，我们就会取得更大的进步。
