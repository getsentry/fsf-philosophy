更新已安装 Windows 的正确方法是什么？

Richard Stallman 著

在计算机领域，敦促用户“升级”到新版本的 Windows（以及其他非自由程序）以便修复“安全性问题”是很常见的。这个结论基于这样的假设：这些程序是诚实的软件，被设计成善待用户的权利。我们并不期望这种情况发生，而且我们知道 Windows 的情况并非如此。因此，我们提出一个不同的建议。

事实上，新版本的 Windows 在发布之前就有漏洞。它们被设计成包含各种恶意功能；参见 “微软的软件是恶意软件。”

较新的 Windows 版本包含附加的恶意功能；它们使微软对用户拥有更大的控制力。这就是不切换到新版本的原因。对于一些用户来说，切换到更新的版本可能需要用金钱回馈微软，甚至购买一台新电脑。我们可不想鼓励这种行为！

因此，我们拒绝通过敦促用户切换到新版本（的操作系统）的行为支持微软；我们并不认为 Windows 无意的缺陷比有意的缺陷更重要。

我们对使用任何版本 Windows 的用户的建议是升级到 GNU/Linux。

在任何版本的 Windows（或任何非 GNU 类系统）上支持 GNU 包不是 GNU 项目核心任务的一部分。我们的口号是“它（GNU 包）在 GNU/Linux 上运行得最好”我们与希望维持这种支持的用户合作，因为在不困难的情况下我们也可以合作。我们没有责任继续这样做，但只要可行且于我们无碍，我们也没有理由停止（在非 GNU 类系统上支持 GNU 包的行为）。
